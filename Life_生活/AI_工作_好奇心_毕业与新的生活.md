# AI,工作,好奇心 - 毕业与新的生活

不知不觉，大学四年的生活已然过去。还记得大一刚刚入学没多久的时候，学校曾有大我一届同学应该是做一些学校布置的任务吧，他们问我，在大学里觉得怎么样。我当时刚毕业年轻气盛，说我觉得大学里要比高中好的多，我可以自学很多之前无法自学的东西。现在想来确是有些过度吹嘘自己了，如果换做是现在的我，我可能就会说"大学可以不用像高中那样紧绷，像停下来休息一下总是有机会"这样的话吧。但是，我认为我确实践行了我所说的话，我在大学完全通过自学学到了很多...

<!--more-->

我接触代码的时间点还算是挺早的，在中学我就在课余时间接触了 C 语言，甚至用他来开发 minecraft 基岩版的一些指令，诸如使用简单的文本输入输出创建一个 `*.mcfunction`，想想那段时光还是很开心的，混迹在各种技术群里，看着各路大佬说着我当时一点也听不懂的话，甚至有时我也会做出一些很孩子气的请求，不过万般尴尬也确实引我入得此途。不过实际上在进入大学之前，我所有的知识也不过是本科 C 语言课程上已有的那些东西罢了。这个博客见证了我成长的过程，自 2023 年建成以来，虽然中间因为一些原因时间戳全部丢失，但也在一定程度上反映了我的学习历程。已经过去这么久了啊。

在这段学习过程当中，我产生了不少的问题。最近这段时间，在秋招千辛万苦获得一个 offer 之后，我启动了不少之前没干完的东西，却发现很多问题，随着知识面上来之后，有了完全不一样的看法。我时常能在做东西以及翻看互联网上新东西获取 idea 的时候发现，我突然能和几年前的自己没解决的问题形成呼应：比如在大一时，我其实对模板元编程仍有困惑之处，最近才发现其可以模拟一个相对较弱的函数式系统，甚至标准后续添加的新特性都从某种程度上利好这一个说法，能和这些东西串起来。

但正如我的 Github Profile 上所写的，我并非是一个 Software Engineer，虽然我即将开始作为一名初级程序员从事这项工作，但我所学习的本科课程可以说是基本与我的工作正交性极高：虽然我所学的通信工程与软件工程同属于电子信息这个超大范畴，但其实两者对电子世界的理解完全位于两极，仅仅只是偶有联系。我认为，相比于纯科班的学生，我确实在学习生涯里加入了噪音，如果我从一开始就选择计算机，或许我能做的更好？

记得大一经由我校校科协的 C++ 组免试题进入到社团这个对我一生可能都有重要影响的一个学生团体，才没有想那么多。非常自然的，凭借着一点点可能先于一部分同龄人的计算机知识，在社团举办的活动中尝到了一些甜头，却在真正进入到此处的时候，感到无力。同样进入 C++ 组的其他朋友，都比我要强许多，我并不能听懂当时在我看来 C++ 一些新的语法(当然现在你让我说精通我也不行)，唯有沉默。不过好在，校科协是一个极具包容性的环境，在技术的氛围之下，一群大佬在一起讨论，我作为小卡拉米也能耳濡目染了。

我们的社团细分了很多组，几乎覆盖了整个电子世界，而软件开发的部分，也是细分到了具体的方向，单纯只是呆在 C++ 组就让我感到困惑，那自然大家都给我创造了不少问题。

而我，是一个极具好奇心的人。

就像我在 C++ 组，大家突然提出"让我们学一下 Qt"和"让我们用 Slint 吧"这样的话题，一顿企划下来，突然有一天框架已经被完成，而我只能够在其中承担一小部分的，基于一个模糊的设计来还原的代码，甚至在绝大多数时候我会得到一句"照着写就行"，这让我感觉不是很好。我不明白整个系统是怎么工作的，甚至也不会有人给我介绍说这个项目有哪些模块组成，要用的函数可能框定在哪一个范围之内。在某天听见大家讨论说，某个学弟只会黑框框，不知道怎么写 UI 程序，我心里一个隔愣，我真的懂怎么写 UI 程序么？当时的我只是会用 UI 框架里预设的一些组件完成一些事情而已，我也没理解为什么曾经看到的包括 Android 自定义组件实则并没有定义什么布局组件的，甚至记得当时看到的资料也没有那种自绘的 UI，我和只会写黑框框没什么区别，唯一的区别就是我站在了前人的肩膀上，在互联网的某个角落点击了下载按钮，安装了一个库而已。

也像是我在学习课堂上的内容时，我总是觉得几门数学课是那么的容易，因为他们的前置条件，defination 是那么的清楚，只要随便一看，便可洞察其中含义。外加我所在的大学的非理科下数学课程本来就难度非常之低。以至于我觉得诸如"电磁场"这样的课程，只要花费某些通信技术十分之一的时间，我就能随便通过考试。而某些实际技术里的概念，通常涉及到巨量的概念相互纠缠，名字写满在一页纸上，也从来没有实现。更加可怕的是，我没有办法像数学物理等课程一样去验证他们，即使我甚至有精力这么做，我也会发现，某些通信技术的现实架构知识只处在企业当中，外部难以模拟，资料也难以获取，管中窥豹相当困难。

不管是什么，我没有任何办法，计算机领域的东西，如果没有资料，在当时没有 AI 的情况下，也只有阅读 Blog 阅读源码这一条路走。而某些学校课程上连大框架都没看懂的东西就真的只能阅读更多的书籍，祈祷有一天能在某个地方看到我想要的东西。好奇心，浪费了我很多时间。

我知道有些朋友的观点与我并不相同，他们并不理解我为什么要这么做，在课余或下班时间就应该出去享受生活，或者写一些别的东西：既然库已经封装好了，就直接用它；既然学校已经教会了公式，用就好了。而且我的成果看上去既不显著，实际上也并没有带来任何的经济效益。如此执着于尽可能搞清楚一切只会搞砸一切。Sure，我承认我正在为捡芝麻丢西瓜支付不小的代价。除了解除我对大框架的认知障碍，满足以下好奇心，其实并没有什么用。我时常发现我费劲心力才终于明白的东西，大家似乎早都知道了。

但其实我的学习方向正是秉持着我的好奇心一步一步生长的，因为我最早会了 C，我接触了 C++，然后发现只能黑框框，于是我知道了 Qt，正式踏上客户端相关领域开发，而在这时候我甚至对前后端一无所知。后面写了一两个程序，因为 C/C++ 在 Windows 开发体验过于之差，转向使用 Linux 作为主力系统。在 Linux 下为了存活下来，系统不会被我使用坏，顺应当时学计算机的同学的学习节奏补充了OS，与计网的知识。紧接着到了最迷茫的时候，在试图比赛来获得保研资格，尽可能提高自己的在校的绩点，以及最后发现各种原因不成可能需要考研中来回摇摆大二大三，这段时间里，我也学习了很多东西，不管是借由通信的知识略微了解了一些数字 IC 设计，对体系结构感兴趣做了一些烂尾的尝试，或是跟着所有 C++ 组以外的同学学习了许多他们所在方向的皮毛，包括前后端、运维、图形学、函数式编程、以及一些编程语言(C#)等一系列技术，其实看了不少东西，做的相对却变少了，但说真的，相比于其他路线可能存在的跳跃性的学习，我真的只是一路顺着思路过来的，如果要偏离这条我实践走过的路线，我还真的不知道怎么学。学习本身其实是一门技术，更是一门艺术，而重视它的人，并不多。

不过说真的，我也不是对技术毫无偏好，比如 AI 就曾是我完全不在意的方向，一来其就业需要极高的学历要求，我达不到。二来我实在是对概率统计没什么兴趣，尤其是网上铺天盖地解释一个我觉得完全没有必要详细解释的梯度下降算法，这在我看来仅仅只是普通微积分的简单应用，实在不值得多言。如今看来是受了互联网低质信息的祸害了。

前面我说我突然能和几年前的自己没解决的问题形成呼应，实际上很多问题我直到现在才终于有了理解的能力。我会思考，我真的做对了么，我可以听到各种群里在讨论什么问题的时候说自己插不上话，我其实很多时候也是插不上话的那个人。虽然我们经常聚在一起聊起开发，好像计算机已经成为了我们的一部分，是我的手，是我的眼，但我们并没有真正了解彼此。

近来，所有社交平台都在讨论 AI Agent，这令我很不舒服，但我却说不上来是什么。今天我因为观看[真理元素关于 2024 年 xz-utils 供应链攻击](https://www.bilibili.com/video/BV1CxcXzuEVY)的时候，好像明白了一些什么：对于 xz-utils 这样一个开源项目来说，这样的攻击防不胜防，但正是因为开源，才有可能最终揪出这样的问题。开源精神十分伟大，而漏洞的发现者 Andres Freund，只是因为在做 PostgreSQL 性能测试时，发现 sshd 占用了 0.5 秒 的额外 CPU 时间，并且登录延迟多了那么一点点，才开始疯狂翻阅系统调用和动态链接过程，最终抓住了这个幽灵。这是一位值得敬佩的软件工程师，实质上 sshd 的问题和他没有任何的关系，没有人会因为 sshd 的延迟而责怪 PostgreSQL，但他却需要潜入现代庞大的软件系统中去找寻那一丝的可能性，这是真正的极客精神吧。我会想，如果我遇到了这样的情况，我是否有能力发现问题，顺着问题追查下去，面对如此优秀的攻击手段，你有能将这个问题完整呈现出来的能力，这本身就相当困难了。当然这一位程序员也并不是等闲之辈，可能视频里只是介绍他是一个普通的员工，但作为全世界称赞的 PostgreSQL 数据库的开发者之一，微软员工，不知道比我们这些真正的普通人强上多少。

不管怎么样，在我看来这个世界不能少了极客精神。在这样一个追求"快速迭代"和"黑盒封装"的时代，大家总是争抢着说最近有一个新的库、新的工具上线，我要尝尝新，却很少见人思考这样的库到底是谁一直在造。当真正的问题发生的时候，近乎执拗的、最纯粹的极客直觉却最终成为了解决问题的方案。大家总是关心结果，却不关心过程。

所以这也是我为什么不是很喜欢 ai，不喜欢讨论 vibe coding，可能本质上就是因为我认为他们是违反好奇心和极客精神的。他们正在试图使用概率统计学的生成工具"一剑破万法"，抹杀思考和学习的艺术。我不反对 vibe coding 本身，我也经常使用它，可在回看 xz-utils 事件后，我突然觉得，AI 提供了一个极其丝滑的自行车，绕过了一切思考的痛苦，未来的开发者可能只剩下"描述需求"的能力，而丧失了"理解实现"的能力。当底层的摩天大楼出现裂缝时，就像 xz-utils 这件事，没人再会知道那是由于一个 m4 宏的问题，还是那 0.5s 的延迟只是偶尔的由于一个硬件漏洞。

我不希望有一天我真的将工具当成知识的终点，这几乎总是让我警铃大作。可是资本垄断正在抹杀极客精神和个人的好奇心，他们正在试图让所有人成为他们的螺丝，并且自从 AI 出现后，似乎连成为螺丝的资格都没有了。他们用全球的已有的知识宝库训练出模型，再把 AI 卖给开发者，甚至作为一个开发者在明知道 OpenClaw 工具在当下具有严重的安全漏洞的时候，还要将其推向大众，赚取利益。他们抹杀了底层路径，想让开发者丧失对底层的解释权。如果你习惯了让 AI 写逻辑，当你遇到 xz 这种级别的攻击时，你除了等待大厂发补丁，没有任何还手之力。AI 带来的不仅是程序员失业率，在裁去初级程序员后，计算机的门槛将会前所未有的提高，在我看来这种情况更有可能导致真正能解决这些问题的新鲜血液发生时代断层。

我不能说我是一个有技术的极客，我只是一个普通的初级程序员，只是恰好喜好探索这些已经”被人抛弃的建造金字塔的技术“而已。实际上堆 UI、接接口、写业务逻辑，才是我可能未来工作的常态。我现在也比较菜，只有对大体逻辑的认识，并不能真正解决类似这样复杂的问题。我也会想，AI 时代其他人想了解皮毛会变得很快，我属于是浪费时间了啊。但我始终认为我对一切有好奇心是十分必要的，尽管这种好奇心即将让我成为一个可能在未来无法供给自己生活的状态，我还是不想抛开。

说来我的大学生活似乎充斥着焦虑、与布满全程的技术话题，就这么走到了尽头。好像离一开始想的，出去走走，别一天到晚做这些事情的那个我相去甚远。以及对自己相当抱歉的是，我用了大学四年去试错，确实没能在通信以及其他电子行业有能推进下去的进展。在这条道路上，我会迷茫，会失落，会觉得自己的能力不足，会孤单，会寂寞，会因为自己还是没能实现一开始谈恋爱的想法而遗憾。不过从此之后，该放下的要放下，而我也只余下这份宝贵的好奇心了。我对我的大学生活在技术之上的进展还是非常满意的，所以继续写下去吧，过去的我能做到，以后的我一定也能做到。或许会支付一定的代价，但我仍然坚定，保持学习，保持好奇。
